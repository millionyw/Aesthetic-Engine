# 个人图片审美偏好引擎 (Aesthetic Preference Engine) - 需求文档

## 1. 业务目标 (Business Objectives)

* **核心任务**：构建一个轻量级、私有化的 AI 模型，学习用户对人物（五官、体态、气质、穿着）的特定审美。
* **效率目标**：通过少于 200 张图片的标注，实现对海量 X (Twitter) 图片的自动化筛选（准确率 > 85%）。
* **应用场景**：离线图片库自动分级、社交媒体流实时/准实时过滤。

---

## 2. 系统功能需求 (Functional Requirements)

### 2.1 数据采集与预处理模块 (ETL Pipeline)

* **输入适配**：支持从本地文件夹或 X API 抓取图片。
* **人体检测与对齐**：自动识别图片中的人物。如果一张图有多个主体，需支持裁剪或焦点识别。
* **多模态特征提取**：
    * 利用预训练大模型（如 **CLIP-ViT-L/14** 或 **DINOv2**）提取全局语义特征。
    * 利用专门的人脸识别模型（如 **InsightFace**）提取五官比例特征。
    * *注：这是为了捕捉你提到的“五官”和“气质”等细粒度属性。*



### 2.2 交互式标注与主动学习 (Active Labeling UI)

* **分级系统**：提供 1-5 分的打分界面，支持快捷键操作。
* **样本推荐算法**：
    * **多样性采样**：在初始阶段，向用户推送风格差异最大的图片。
    * **不确定性采样**：在训练阶段，优先推送模型评分在 2.5-3.5 分之间（即模型“纠结”的）图片。


* **数据增强**：对用户打高分的图片自动进行镜像、裁剪增强，以扩充小样本数据集。

### 2.3 模型训练与推理模块 (Model Core)

* **架构设计**：采用 **Linear Probing** 方案。冻结大模型骨干网络，仅训练末端的分类器/回归器（MLP）。
* **输出定义**：
    * `Score`: 1.0 - 5.0 的浮点数。
    * `Embedding`: 该图片的审美特征向量，用于后续的相似度检索。



---

## 3. 非功能性需求 (Non-functional Requirements)

| 维度 | 要求 |
| --- | --- |
| **隐私性** | **100% 本地化**。用户的审美偏好属于极度隐私，模型训练与推理不得上传云端。 |
| **响应速度** | 离线筛选速度需达到 > 10 张/秒（在消费级 GPU 或 Mac M系列芯片上）。 |
| **可解释性** | 系统应尝试输出“预测权重”，例如：这张图得高分是因为“穿着”特征还是“五官”特征。 |

---

## 4. 系统逻辑架构 (System Architecture)

1. **感知层 (Perception)**: 使用开源的 Vision Transformer 提取高维特征向量 。
2. **映射层 (Mapping)**: 训练一个函数 ，拟合用户的非线性审美。
3. **反馈层 (Feedback)**: 用户纠正错误评分后，触发增量训练（Incremental Learning）。

---

## 5. 验证与测试标准 (Acceptance Criteria)

### 5.1 定量指标 (Quantitative)

* **Top-K 准确率**：模型预测的前 10% 高分图片，用户认可度（4-5分）比例应超过 80%。
* **标注效率**：用户只需标注 100-200 张图片，模型 Loss 趋于平稳。

### 5.2 定性指标 (Qualitative)

* **鲁棒性**：模型不应因为换了背景颜色或滤镜就对同一人物的气质判断产生剧烈偏差。
* **冷启动表现**：在用户仅标了 10 张图时，系统应能展示出基本的审美倾向（如：开始避开用户明显讨厌的风格）。
